
# 高效的ADSR包络算法设计

L-MODEL团队 赤狐

## 简介

众所周知，ADSR包络是合成器里最常见的控制信号之一。一个音符按下之后，声音从Attack上升到峰值，再经过Decay回落到Sustain电平，最后在松键时进入Release并逐渐消失。

这件事听起来非常简单，但是真正塞进逐采样音频处理里，就会变得有点麻烦。传统做法通常会写一个状态机：当前是Attack还是Decay？这一段有没有走完？松键时要不要切到Release？当前曲线是线性、指数，还是某种解析函数？如果还要支持实时调参，就得在状态切换、曲线求值和边界修正之间来回打补丁。

这些计算单独看都不贵，但是复音一多、调制一密，包络就会变成一个非常烦人的热路径。

这篇博客介绍一种算法：把ADSR拆成几段一阶响应曲线，并把曲线解析式整理成递推形式。运行时不再反复计算指数函数，也不再写复杂状态机，而是用4段系数表驱动包络。每个采样只需要一次乘加、一次位置递推，就能得到高效而且曲线形状可调的ADSR。

## 一阶曲线的玩法

我们先不急着讨论ADSR，先看一个更小的问题：

如果我想让一个值从$y_0$走到$y_1$，并且希望它不是单纯直线，而是可以调节弯曲程度的曲线，应该怎么做？

一个很自然的选择是指数曲线。设段内采样位置为$x$，这一段总长度为$\mathrm{len}$，其中$x\in[0,\mathrm{len}]$。我们希望曲线满足：

$$
f(0)=y_0,\quad f(\mathrm{len})=y_1
$$

再给它一个形状参数$p$。当$p=0$时，它退化成线性；当$p$不为0时，它变成指数弯曲。经过一番整理，可以得到：

$$
f(x)=y_0 e^{\frac{x}{\mathrm{len}}p}
+(y_1-y_0e^p)
\frac{e^{\frac{x}{\mathrm{len}}p}-1}{e^p-1}
$$

大家可以检查一下端点。令$x=0$，指数项为$1$，后面的分式为$0$，所以$f(0)=y_0$。令$x=\mathrm{len}$，后面的分式为$1$，于是前面的$y_0e^p$刚好被抵消，最后得到$f(\mathrm{len})=y_1$。

也就是说，这条曲线不管怎么弯，端点都是精确的。

图片1：不同$p$下，从0到10的一阶曲线形状。

desmos:https://www.desmos.com/calculator/xbzo8bdvx0

其中$p$控制曲线的弯曲方向。对于从0到1的Attack来说，正的$p$会让曲线前面更慢、后面更快；负的$p$会让曲线前面更快、后面更慢。当$p$非常接近0时，直接使用线性插值即可：

$$
f(x)=y_0+(y_1-y_0)\frac{x}{\mathrm{len}}
$$

这样做还有一个好处：可以避开$e^p-1$在$p$接近0时带来的数值问题。

## 从解析求值到递推求值

到这里为止，我们已经有了一条好用的曲线。但是如果每个采样都直接算一次$f(x)$，就还是要反复调用指数函数。对于包络这种每个音符、每个采样都要跑的东西来说，这就不太便宜。

所以关键问题变成了：能不能把这条曲线改写成递推形式？

假设这一段长度为$\mathrm{len}$个采样。每前进一个采样，段内位置就增加：

$$
\Delta x=1
$$

于是指数项会乘上一个固定比例：

$$
e^{\frac{x+\Delta x}{\mathrm{len}}p}
=e^{\frac{x}{\mathrm{len}}p}\cdot e^{p/\mathrm{len}}
$$

这就意味着，整条曲线可以写成一个非常简单的一阶递推：

$$
y[n+1]=s\cdot y[n]+k
$$

其中：

$$
s=e^{p/\mathrm{len}}
$$

而为了让递推曲线仍然精确到达$y_1$，偏置项$k$可以整理为：

$$
k=(y_1-y_0e^p)
\frac{s-1}{e^p-1}
$$

注意这里的$k$不是随时间变化的，它在参数更新时预先算好即可。于是运行时就不需要再碰指数函数了，只需要：

```cpp
y = s * y + k;
```

当$p$接近0时，这个递推也退化为线性形式：

$$
s=1,\quad k=\frac{y_1-y_0}{\mathrm{len}}
$$

所以这套算法本质上不是“每次求曲线值”，而是“先把曲线设计成一阶系统，然后让一阶系统自己往前走”。

## 组合Attack、Decay和Release

有了这个一阶曲线，ADSR就可以拆成三条真正需要运动的曲线：

1. Attack：从$0$走到$1$。
2. Decay：从$1$走到$\mathrm{sustain}$。
3. Release：从当前值衰减到$0$。

在代码里，这个过程被封装成了一个`SetSegment`函数：

$$
\mathrm{SetSegment}(y_0,y_1,\mathrm{len},p,i)
$$

其中$y_0$表示这一段曲线的起点，$y_1$表示这一段曲线的终点，$\mathrm{len}$表示这一段持续多少个采样，$p$表示曲线弯曲程度，$i$表示这组系数要写入第几段系数表。

函数内部会根据这些参数预先算出这一段的一阶递推系数：

$$
s_i,\quad k_i,\quad r_i
$$

也就是说，`SetSegment`并不是直接生成一整段包络采样点，而是把“从哪里来、到哪里去、走多快、怎么弯”这几个信息，压缩成运行时可以快速递推的三个系数。

Attack和Decay都可以直接使用上一节的一阶曲线。Attack调用：

$$
\mathrm{SetSegment}(0,1,\mathrm{attackLen},\mathrm{attackShape})
$$

Decay调用：

$$
\mathrm{SetSegment}(1,\mathrm{sustain},\mathrm{decayLen},\mathrm{decayShape})
$$

这两段都有明确的起点、终点和长度，所以非常适合用可调$p$的指数曲线来描述。

Release稍微特殊一点。松键可能发生在任何时刻：可能Attack还没走完，也可能Decay中间，也可能已经停在Sustain。因此Release的起点不是固定的。这里使用一个更便宜的做法：把Release设计成单纯的一阶衰减：

$$
y[n+1]=s_\mathrm{rel}\cdot y[n]
$$

其中：

$$
s_\mathrm{rel}=e^{-6.90775527898/\mathrm{releaseLen}}
$$

$6.90775527898$其实就是$\ln(1000)$。也就是说，经过$\mathrm{releaseLen}$个采样后，包络会衰减到原来的$0.001$，也就是RT60意义上的“基本听不见”。这样Release就不需要知道自己的起点是多少，当前$y$是多少，它就从哪里自然衰减下去。

## 用4段系数表代替状态机

到这里还剩最后一个问题：怎么让ADSR在Attack、Decay、Sustain、Release之间切换？

常规做法会写一堆状态判断。不过这里可以更偷懒一点：直接把四段都放进表里。

```cpp
// 0: release 1: attack 2: decay 3: sustain
float s[4] = { 0 }, k[4] = { 0 }, r[4] = { 0 };
float pos = 0;
float y = 0;
```

其中$s$和$k$就是递推系数，$r$表示这一段每个采样推进多少位置。`pos`同时承担两个作用：它的整数部分表示当前段，它的小数部分表示当前段内部的归一化进度，也就是$x/\mathrm{len}$。

于是每个采样的更新可以写得非常短：

```cpp
int idx = pos;
y = s[idx] * y + k[idx];
pos += r[idx];
if (pos > 3.0f) pos = 3.0f;
```

Attack段的`pos`从$1$开始，`r=1/\mathrm{attackLen}$，走完后自然进入$2$，也就是Decay。Decay段从$2$开始，走完后自然进入$3$，也就是Sustain。Sustain段的$r=0$，所以会停在那里。

Release段被放在$0$。松键时只需要：

```cpp
pos = 0.0f;
```

因为Release本身是当前位置上的指数衰减，所以不需要重算起点，也不需要根据Attack或Decay的中间状态做额外分支。

按键时则进入Attack：

```cpp
pos = 1.0f;
y = 0.0f;
```

于是整个“状态机”就被压缩成了一个浮点位置和三张4段系数表。

## 实时调参时的处理

还有一个细节很有意思：参数变化时，Attack和Decay的曲线系数会重新计算。如果此时包络正处在Attack或Decay中，直接继续使用旧的$y$，可能会和新曲线的当前位置对不上。

所以可以根据`pos`的小数部分，重新解析求值一次当前段：

```cpp
int idx = pos;
float posFrac = pos - idx;
if (idx == 1) y = EvalSegment(0.0f, 1.0f, attShape, posFrac);
else if (idx == 2) y = EvalSegment(1.0f, susV, decShape, posFrac);
else if (idx == 3) y = susV;
```

这里的`posFrac`就是$x/\mathrm{len}$，所以它可以直接传给`EvalSegment`做一次低频率的解析求值。

注意，这里的解析求值只发生在参数更新时，不发生在逐采样热路径里。也就是说，我们把贵的计算挪到了低频率事件里，把便宜的递推留给音频线程。

这就是整个算法最核心的设计意图：设计时多算一点，运行时少算很多。

## 算法流程

1. 根据采样率把毫秒参数转换成采样长度：$\mathrm{len}=\mathrm{sampleRate}\cdot\mathrm{ms}/1000$。
2. 对Attack和Decay调用`SetSegment`，预先计算每段的$s$、$k$、$r$。
3. 对Release使用RT60形式的一阶衰减，预先计算$s_\mathrm{rel}$，并令$k=0$、$r=0$。
4. 对Sustain令$s=1$、$k=0$、$r=0$，让包络保持当前值。
5. 每个采样根据`pos`查表，执行$y=s_i y+k_i$，再用$r_i$推进`pos`。
6. Note On时跳到Attack，Note Off时跳到Release。

## 具体实现

大家可以去看L-MODEL倾情制作的Dirtynth合成器包络源码，里面有具体实现：

https://github.com/hiirofox/Dirtynth/blob/master/Source/dsp/Envelope.h

这个实现最有意思的地方不在于用了多复杂的数学，而在于把数学整理成了很适合工程落地的形状。Attack和Decay仍然保留可调曲线，Release可以从任意当前值自然衰减，Sustain不需要额外计算。而逐采样更新只剩下一次乘加和一次查表推进。

对于合成器来说，这种算法尤其舒服：复音数量越多，包络更新越频繁，热路径越便宜就越有价值。最后得到的不是一个“功能很多但很重”的ADSR，而是一个曲线形状可控、实时调参友好、计算量极低的ADSR。
